# AGENTS.md

## Dev Commands

```bash
bun dev              # Dev server on localhost:3001
bun run build        # Full build + generate dist/client/index.html
bun run lint         # ESLint (no typecheck step configured)
bun run format       # Prettier (printWidth 100, singleQuote false)
```

## Deployment

- **Cloudflare Workers** (primary): `src/server.ts` is the entry point (configured in `wrangler.jsonc`). Uses `nodejs_compat` flag.
- **Vercel** (static SPA fallback): `vercel.json` outputs `dist/client`, rewrites all routes to `/index.html`. Post-build script `scripts/generate-html.js` runs the SSR server to produce a static `dist/client/index.html`.

## Build Config

- **`vite.config.ts`** uses `@lovable.dev/vite-tanstack-config`. Do NOT add plugins manually (React, Tailwind, TanStack, tsConfigPaths, Cloudflare are already bundled).
- Server entry explicitly set via `tanstackStart: { server: { entry: "server" } }` (resolves to `src/server.ts`).

## Project Structure

- **SSR framework**: `@tanstack/react-start`. Server entry wraps TanStack's server entry with h3 error recovery (`src/server.ts` + `src/start.ts`).
- **Router**: TanStack Router, file-based (`src/routes/`). Route tree auto-generated by `@tanstack/router-plugin`.
- **Auth**: Supabase Auth via React context (`src/features/auth/hooks/use-auth.tsx`). Uses a module-level singleton promise to avoid duplicate `getSession()`.
- **Maps**: Leaflet + react-leaflet (`src/features/map/`).
- **Data**: Supabase for venues, matches, RSVPs. Google Places API proxied through a Supabase Edge Function (`supabase/functions/google-places/`).
- **Push notifications**: Firebase Cloud Messaging (FCM). SW file at `public/firebase-messaging-sw.js`.
